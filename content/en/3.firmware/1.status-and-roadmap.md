# Firmware

::warning
Firmware for the full Keero platform is still coming soon.
::

The documentation is ready for the firmware section now, but the root firmware stack is not yet published as a complete, stable application. What exists today is an early `tracks` module firmware with enough detail to document the current direction.

## Current Implemented Firmware Work

Repository path:

```text
keero-firmware/modules/tracks/
```

### PlatformIO target

```ini
[env:esp32-c3-devkitc-02]
framework = arduino
board = esp32-c3-devkitc-02
monitor_speed = 115200
upload_speed = 115200
board_build.filesystem = spiffs
board_build.partitions = config/partitions_4mb_c3.csv
```

### Existing responsibilities in code

- UART command parsing in `src/main.cpp`
- motor PWM drive in `lib/motion_control`
- WS2812 light effects in `lib/light_control`
- battery voltage monitoring and charge-state driven light overrides in `lib/power_management`

## Tracks Module Behavior Today

The current firmware parses commands over `Serial1` on:

- TX: GPIO `10`
- RX: GPIO `18`
- Baud rate: `115200`

Supported command patterns in the current code:

- `x<float> y<float>` or `x<float>y<float>` for differential motion control
- `w<float>` to set a light mode
- `c<float>` to adjust motor speed balancing coefficients
- `d<float>` to trigger the dance routine

### PWM motor pins

- left motor A: GPIO `4`
- left motor B: GPIO `5`
- right motor A: GPIO `6`
- right motor B: GPIO `7`

### RGB lighting

- WS2812 GPIO: `0`
- pixel count: `6`

Implemented light modes include:

- charging breath
- low power alert
- always on
- blink
- slow white breath
- fast white breath
- flowing rainbow
- show mode
- sleep

### Battery monitor

- battery ADC pin: GPIO `3`
- charge detect pin: GPIO `2`
- low battery threshold in code: `20%`
