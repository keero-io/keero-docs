# Modules and Mechanics

## Module Strategy

The hardware repository is intentionally split so the mainboard remains the reusable brain while modules add mobility, docking, or future sensors.

The repository currently contains:

- `modules/dock/`
- `modules/tracks/`

## Dock Module

The dock is described in the repository as a charging and development base. The mainboard already supports this with pogo-based connectivity and exposed power/UART lines.

## Tracks Module

The tracks module currently contains mechanical models rather than finished electronics:

- `ESP-SparkTank_Body_top.STL`
- `ESP-SparkTank_Body_btm.STL`
- `ESP-SparkTank_driver_wheel.STL`
- `ESP-SparkTank_idle_wheel.STL`
- `ESP-SparkTank_idle_wheel_cover.STL`

This lines up with the existing firmware work in `keero-firmware/modules/tracks`, where motor control, lighting, and battery monitoring are already under development.

## Integration Points

The module system is expected to use:

- pogo pins for docking and power
- UART for module communications
- shared power rails from the mainboard
- user-accessible headers for quick iteration during hardware bring-up

## Mainboard Enclosure Assets

The mainboard folder also includes printable shell components and a full STEP model, which is useful both for enclosure checks and collision verification.
